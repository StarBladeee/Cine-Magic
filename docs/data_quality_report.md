# 豆瓣电影数据集质量验收报告

## 概述

对 `data/douban/` 目录下的三个原始 CSV 文件进行了全面质量检查，识别并修复了缺失值问题。

---

## 一、movies.csv（全量 140,502 部电影）

### 1.1 DOUBAN_SCORE（豆瓣评分）

| 状态 | 数量 | 占比 |
|------|------|------|
| 有真实评分 | 25,968 | 18.5% |
| 评分缺失（原记 0.0） | 114,534 | 81.5% |

**问题**：`DOUBAN_SCORE=0.0` **既是**"数据缺失占位符"**也是**"可能的真实 0 分"。全量数据中 81.5% 的电影评分为 0.0，其中既包括真正的冷门电影（无评分无人气），也包括大量高人气电影（投票数>50,000）因采集 bug 缺失评分。

**修复**：将 `DOUBAN_SCORE=0.0` 全部替换为空字段（CSV 为空，pandas 读为 `NaN`），使"缺失"与"零值"彻底分离。评分人数同理。

### 1.2 DOUBAN_VOTES（豆瓣评分人数）

| 状态 | 数量 | 占比 |
|------|------|------|
| 有真实数据 | 38,562 | 27.4% |
| 数据缺失（原记 0.0） | 101,940 | 72.6% |

**问题**：与 DOUBAN_SCORE 相同，`0.0` 表示未采集到的实际缺失（这些电影在豆瓣上可能确实有评分人数，但数据集中未记录）。

**修复**：同上，`0.0` → 空字段。

### 1.3 高投票但评分缺失的典型案例

全量数据中共 183 部投票数超 20 万但评分为空的电影（包括肖申克的救赎、楚门的世界等），已在 `top200_fixed.csv` 中通过爬虫修复了 91 部。

### 1.4 其他字段缺失统计

| 字段 | 缺失数 | 缺失率 | 评价 |
|------|--------|--------|------|
| GENRES | 4,050 | 2.9% | 基本完整 |

| REGIONS | 4,001 | 2.8% | 基本完整 |

| IMDB_ID | 27,246 | 19.4% | 部分缺失 |

**评价**：核心分类字段（类型、年份、地区）完成度很高，但描述性字段（剧情、演员、导演）缺失严重。对于基于协同过滤的推荐模型影响不大，但内容召回需要依赖这些文本特征。

### 1.5 类型数据

55 种类型标签，有中英混杂问题（如"剧情"和"劇情 Drama"是同一类型），需后续归一化处理。

## 三、Top200 数据修复结果

### 3.1 原始 top200.csv

- 200 部电影中 **91 部**（46%）的 DOUBAN_SCORE 为 0.0 但 DOUBAN_VOTES > 5 万
- 均为豆瓣高人气电影，评分缺失由采集 bug 导致，非真实 0 分

### 3.2 爬虫修复

| 来源 | 数量 |
|------|------|
| 爬虫成功抓取 | 91 部 |
| 原始数据正常保留 | 108 部 |

## 五、待处理问题

1. **114,534 部电影评分仍为空**：其中 183 部是热门电影（投票>200k），需要后续通过爬虫逐步修复；其余是真正的冷门电影
2. **类型中英混杂**：55 种类型中"剧情"和"劇情 Drama"实为同一类型，需归一化
3. **DIRECTORS/ACTORS 缺失 50%**：全量数据中一半电影没有导演/演员信息
4. **V字仇杀队**：豆瓣屏蔽，需从其他来源（如 OMDb/TMDB）获取评分
5. **ratings 评分右偏**：用户倾向于打高分，评估时 `rating_threshold=3.5` 可能需要调高至 4.0 才能更好区分"真正喜欢"
